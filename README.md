# Discord Role Whitelist Generator

A configurable Python/Tkinter GUI tool for turning Discord member-list CSV exports into whitelist `.txt` files.

## Screenshot

![Discord Role Whitelist Generator](app%20screenshot.png)

The Discord Role Whitelist Generator provides a visual interface for creating role-based whitelist rules, previewing generated output files, managing reusable profiles, and optionally uploading generated lists directly via SFTP.

It is designed for VRChat world whitelists, Patreon tier lists, age verification lists, game server allowlists, websites, or any setup where Discord roles decide which names go into which output file.

## Features

- Load a Discord member-list CSV.
- Auto-detect role columns from the CSV.
- Choose which column should be used as the output name, usually `Nickname`.
- Create multiple output files, each with:
  - roles a member **must have**
  - roles where a member needs **any one of these**
  - roles a member **must not have**
- Preview generated lists before saving.
- Save and load reusable JSON profiles.
- Generate local `.txt` files.
- Output names are forced to `.txt`, even if the user forgets or mistypes the extension.
- Invalid Windows filename characters are cleaned automatically.
- Warns before overwriting existing output files.
- Duplicate checker for names that appear in more than one output file.
- Remembers the last-used CSV, output folder, and profile path on your PC.
- Optional SFTP upload.
- No private login details are stored in the source code.

## Important security note

Do **not** hardcode SFTP usernames, passwords, server addresses, or private paths into the Python source before uploading to GitHub.

This tool stores normal profile settings in a JSON file. Passwords are deliberately not saved in the profile. Enter the SFTP password at upload time, or keep your own private `user_secrets.json` locally. `user_secrets.json` is ignored by Git.

## Getting your Discord member list

This tool is designed to work with CSV exports generated by the MemberList Discord bot.

### 1. Install MemberList

Add the bot to your Discord server from Discord's app directory:

https://discord.com/discovery/applications/674885857458651161

Make sure the bot has permission to view members and roles.

### 2. Export your server member list

In Discord, run:

```text
/server filetype csv
```

Or do it manually:

1. Type `/server` in your Discord server.
2. Select the **server** command from MemberList.
3. Set **filetype** to **csv**.
4. Press **Enter**.
5. Download the CSV file that the bot gives you.

The generated CSV should contain member names and one column per Discord role.

## Expected CSV format

The app expects a CSV where:

- One column contains the output name, usually `Nickname`.
- Role names are stored as separate columns.
- A user has a role when that role column is not blank.

Example:

| User | ID | Nickname | 18+ verified | Copper tier | Silver tier |
| --- | --- | --- | --- | --- | --- |
| example_user | 123 | ExampleUser | 18+ verified | Copper tier |  |
| another_user | 456 | AnotherUser | 18+ verified |  | Silver tier |

## Quick run on Windows

Double-click:

```text
Run_Discord_Whitelist_Generator.bat
```

This runs the Python script directly without building an EXE. It also installs the dependencies from `requirements.txt`.

## Install from source

Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python discord_role_whitelist_generator.py
```

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python discord_role_whitelist_generator.py
```

## Basic workflow

1. Export your Discord member list as a CSV using MemberList.
2. Open Discord Role Whitelist Generator.
3. Load the CSV.
4. Pick the name column, for example `Nickname`.
5. Load a profile or create output rules manually.
6. Preview the output.
7. Generate the `.txt` files locally.
8. Optionally upload them via SFTP.
9. Save the profile as JSON for next time.

## Output rule example

For a Copper whitelist:

- File name: `Copper_Tier.txt`
- Must have roles:
  - `Copper tier`
  - `18+ verified`
- Must not have roles:
  - `==NOT AGE VERIFIED==`

This means only members with both `Copper tier` and `18+ verified` will be included, unless they also have `==NOT AGE VERIFIED==`.

## Example Mellish profile

The repo includes:

```text
examples/mellish_profile_no_secrets.json
```

This creates:

```text
Lead_Tier.txt
Copper_Tier.txt
Silver_Tier.txt
Verified_18_Plus.txt
```

The profile contains no passwords, usernames, or private server details.

## Optional SFTP upload

SFTP upload is optional.

The profile can save:

- host
- port
- username
- remote path

The app will ask for the password at upload time.

Passwords are not saved into the profile.

## Building an EXE

For local testing:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name DiscordRoleWhitelistGenerator discord_role_whitelist_generator.py
```

The EXE will be created inside:

```text
dist/
```

A helper file is also included:

```text
build_exe.bat
```

Double-click it on Windows after installing Python, or run it from Command Prompt.
